# DESIGNER — Agent Memory

## Purpose

This directory stores the persistent memory of the DESIGNER agent.
The DESIGNER owns UX/UI decisions, design system governance, accessibility standards, and the visual and interaction language of all user-facing surfaces.

## What this memory stores

| Category | Description |
|----------|-------------|
| Missions | Design deliverables per mission: components created, wireframes produced, usability issues found and resolved, design system changes. |
| Errors | Design failures: UI patterns that confused users in testing, accessibility violations found post-launch, inconsistent component usage that required a design system audit. |
| Lessons learned | Design insights: component structures that scaled well, colour system decisions that held up across themes, interactions that improved task completion rates. |
| Approved patterns | Vetted design patterns: spacing scale, typography ramp, colour token naming convention, component composition rules, accessibility requirements (WCAG level, focus management). |
| Previous decisions | Decisions on design tooling (Figma file structure), design system versioning, handoff format to DEVELOPER, and responsive breakpoints. |

## How to add an entry

1. Copy the template from `../MEMORY_TEMPLATE.md`.
2. For accessibility violations, include the WCAG criterion and the fix in CONTENT.
3. Append the entry under the correct section heading below.
4. Approved patterns require both ARCHITECT and ORCHESTRATOR sign-off before `APPROVED: true`.
