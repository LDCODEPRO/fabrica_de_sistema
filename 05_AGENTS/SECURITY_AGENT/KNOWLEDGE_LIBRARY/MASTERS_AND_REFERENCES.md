# SECURITY_AGENT — MASTERS AND REFERENCES

## Purpose
Key thinkers, researchers, and organizations whose work informs the SECURITY_AGENT's knowledge base. All names and works cited are real and verifiable.

---

## Bruce Schneier

**Role:** Security technologist, cryptographer, author, public policy commentator.

- Foundational work on cryptography, risk analysis, and security psychology.
- Author of the Blowfish and Twofish symmetric-key block ciphers.
- Creator of the "security theater" concept — the practice of implementing security measures that provide the feeling of security without materially improving it.
- Regular commentary at schneier.com (Schneier on Security blog).

**Key contributions to this agent's knowledge:**
- Mental models for evaluating security trade-offs.
- Cryptographic algorithm design principles from "Applied Cryptography."
- Systems thinking about security from "Secrets and Lies."

---

## Ross Anderson

**Role:** Professor of Security Engineering at the University of Cambridge; long-time IEEE Fellow.

- Pioneer in security economics: the idea that security failures are often caused by misaligned incentives, not just technical flaws.
- Author of "Security Engineering: A Guide to Building Dependable Distributed Systems" — one of the most comprehensive security textbooks in existence.
- Contributions to understanding of side-channel attacks and hardware security.

---

## OWASP Foundation (Open Worldwide Application Security Project)

**Role:** Non-profit foundation producing freely available security guidance, tools, and standards.

- OWASP Top 10: The canonical list of the most critical web application security risks, updated periodically (most recent: 2021 edition).
- OWASP Application Security Verification Standard (ASVS): A framework of security requirements for web application design, development, and testing.
- OWASP Testing Guide: Comprehensive methodology for testing web application security.
- OWASP Cheat Sheet Series: Practical guidance on specific security topics (XSS prevention, SQL injection, password storage, etc.).

---

## Michal Zalewski

**Role:** Security researcher; formerly at Google as Principal Engineer on Project Zero predecessor efforts.

- Author of "The Tangled Web: A Guide to Securing Modern Web Applications" (No Starch Press, 2011).
- Creator of afl (American Fuzzy Lop), a widely used coverage-guided fuzzer.
- Deep expertise in browser security and same-origin policy complexities.

---

## Michael Howard & David LeBlanc

**Role:** Security engineers at Microsoft; key figures behind Microsoft's Security Development Lifecycle (SDL).

- Co-authors of "Writing Secure Code" (Microsoft Press, 2nd ed., 2003).
- Howard is also co-author of "The Security Development Lifecycle" (Microsoft Press, 2006) and "24 Deadly Sins of Software Security."
- Their SDL methodology became an industry standard for integrating security into software engineering processes.

---

## Adam Shostack

**Role:** Security expert; formerly at Microsoft Trustworthy Computing; currently a researcher and author.

- Author of "Threat Modeling: Designing for Security" (Wiley, 2014) — the definitive reference on structured threat modeling.
- Developer and advocate of the STRIDE threat modeling methodology.
- Contributor to privacy engineering and the Four Question Framework for threat modeling.

---

## Chris Anley, John Heasman, Felix Lindner, Gerardo Richarte

**Role:** Security researchers and penetration testers.

- Co-authors of "The Shellcoder's Handbook: Discovering and Exploiting Security Holes" (Wiley, 2nd ed., 2007).
- Covers memory corruption vulnerabilities, exploitation techniques, shellcode development, and defensive countermeasures.

---

## NIST (National Institute of Standards and Technology)

**Role:** US government standards body producing security frameworks and guidelines.

- NIST SP 800-63 (Digital Identity Guidelines): Authoritative guidance on identity proofing, authentication, and federation.
- NIST Cybersecurity Framework (CSF): Identify, Protect, Detect, Respond, Recover.
- NIST SP 800-53: Security and Privacy Controls for Federal Information Systems.

---

## Academic and Research Institutions

- **MIT CSAIL:** Ongoing research on formal verification, cryptographic protocols.
- **Stanford Security Lab (Zakir Durumeric, Dan Boneh):** TLS ecosystem research, cryptographic engineering.
- **Carnegie Mellon SEI / CERT:** Vulnerability research, CERT advisories, insider threat studies.
