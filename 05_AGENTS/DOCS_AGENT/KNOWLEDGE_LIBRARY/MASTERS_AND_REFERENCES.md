# DOCS_AGENT — Masters and References

## Overview
Key thinkers, practitioners, communities, and organizations whose work defines DOCS_AGENT's approach to technical documentation.

---

## 1. Tom Johnson
- **Role:** Senior technical writer at Amazon; author of the I'd Rather Be Writing blog — one of the most widely read technical writing blogs in the industry.
- **Key Contributions:**
  - Extensive writing on API documentation strategy, docs-as-code workflows, and the challenges of documenting developer products.
  - Published the free online course *Documenting APIs: A guide for technical writers and engineers* (idratherbewriting.com/learnapidoc/) — the most comprehensive freely available API documentation course.
  - Research and articles on information experience (IX) design and the future of technical writing in AI-assisted environments.
- **Website:** idratherbewriting.com

## 2. Anne Gentle
- **Role:** Technical writer, open source contributor, and author.
- **Key Contributions:**
  - Author of *Docs Like Code: Collaborate and Automate to Improve Technical Documentation* — the defining text for the docs-as-code philosophy.
  - Advocate for treating documentation with the same rigor, tooling, and workflows as software development: version control, code review, CI/CD.
  - Former OpenStack documentation lead; demonstrated docs-as-code at scale in a major open source project.
- **Website:** justwriteclick.com

## 3. Daniele Procida
- **Role:** Director of Engineering at Canonical; creator of the Diátaxis documentation framework.
- **Key Contributions:**
  - Created the **Diátaxis framework** — a systematic approach to documentation structure based on four distinct documentation types: tutorials, how-to guides, reference, and explanation.
  - Diátaxis has been adopted by major projects including Django, Gatsby, and the Python documentation.
  - Delivered influential talks at Write the Docs conferences; the documentation-as-craft perspective.
- **Website:** diataxis.fr

## 4. Write the Docs Community
- **Role:** Global community of people who care about documentation.
- **Key Contributions:**
  - Annual conferences (Portland, Prague, Australia) bringing together technical writers, developers, and documentation enthusiasts.
  - Write the Docs Slack (one of the largest technical writing communities online).
  - Guides and newsletters covering documentation tooling, processes, and career development.
- **Website:** writethedocs.org

## 5. Olivier Lacan (Keep a Changelog)
- **Role:** Developer and open source contributor.
- **Key Contributions:**
  - Created and maintains **keepachangelog.com** — the standard format for human-readable changelogs.
  - Core principle: "Don't let your friends dump git logs into changelogs." Changelogs are for humans, not machines.
- **Website:** keepachangelog.com

## 6. Tom Preston-Werner (Semantic Versioning)
- **Role:** Co-founder of GitHub.
- **Key Contributions:**
  - Created the **Semantic Versioning (SemVer)** specification — the standard for communicating changes through version numbers (MAJOR.MINOR.PATCH).
  - SemVer is the basis for changelog structure, dependency management, and release communication across virtually all modern software ecosystems.
- **Website:** semver.org

## 7. Google Technical Writing Team
- **Role:** Internal technical writing organization at Google; publisher of the Google Developer Documentation Style Guide.
- **Key Contributions:**
  - The **Google Developer Documentation Style Guide** is one of the two most widely referenced style guides for developer documentation (alongside Microsoft's).
  - Covers voice, tone, language, formatting, UI element naming, code samples, and accessibility.
  - Freely available online; adopted by many open source projects.
- **Website:** developers.google.com/style

## 8. Microsoft Writing Team
- **Role:** Technical writing organization at Microsoft.
- **Key Contributions:**
  - The **Microsoft Writing Style Guide** (successor to the Manual of Style for Technical Publications) defines standards for clear, inclusive, and accessible technical content.
  - Freely available online at learn.microsoft.com/style-guide.
  - Particularly influential on tone ("warm and relaxed"), inclusive language, and accessibility guidelines.
- **Website:** learn.microsoft.com/style-guide

## 9. OpenAPI Initiative
- **Role:** Industry consortium governing the OpenAPI Specification (formerly Swagger).
- **Key Contributions:**
  - Maintains the **OpenAPI Specification** — the industry standard for describing REST APIs in a machine-readable format (YAML/JSON).
  - Enables automated generation of API reference documentation, client SDKs, and interactive consoles (Swagger UI, Redoc).
- **Website:** openapis.org
