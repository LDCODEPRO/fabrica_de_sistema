# TOOLS AND STANDARDS — ANALYST AGENT

## Collaboration and Requirements Tools

### Miro
**Type**: Online collaborative whiteboard  
**URL**: https://miro.com  
**Use Cases**:
- Event Storming workshops (pre-built templates available)
- Domain storytelling
- User journey mapping
- Affinity diagramming for stakeholder feedback
- Requirements workshops with remote teams

**Key Features**:
- Infinite canvas with sticky note, shape, and connector tools
- Real-time collaboration for distributed teams
- Templates for Event Storming, user story mapping, customer journey maps
- Integration with Jira (sync boards to tickets)

---

### FigJam (Figma)
**Type**: Collaborative whiteboard (optimized for design and product teams)  
**URL**: https://www.figma.com/figjam  
**Use Cases**:
- User story mapping
- Stakeholder mapping and journey flows
- Retrospective facilitation
- Rapid requirements sketching linked to Figma design files

---

### Confluence (Atlassian)
**Type**: Team wiki and documentation platform  
**URL**: https://www.atlassian.com/software/confluence  
**Use Cases**:
- Requirements specifications (SRS, BRD, PRD)
- Decision logs and meeting notes
- Business rules documentation
- Requirements traceability matrices
- Linked to Jira tickets for requirements traceability

**Key Features**:
- Page templates for requirements documents
- Inline commenting and review workflows
- Macros for Jira integration (live ticket status in docs)

---

### Jira (Atlassian)
**Type**: Issue tracker and backlog management  
**URL**: https://www.atlassian.com/software/jira  
**Use Cases**:
- User story and acceptance criteria documentation
- Backlog management and prioritization
- Requirements traceability via issue links
- Definition of Done tracking

---

### Cucumber
**Type**: BDD test automation framework  
**URL**: https://cucumber.io  
**Use Cases**:
- Writing and executing Gherkin-format acceptance criteria
- Bridging between business specifications and test automation
- Living documentation generation

## Standards

### ISO/IEC/IEEE 29148:2018 — Systems and Software Engineering: Requirements Engineering
**Organization**: ISO / IEC / IEEE  
**Description**: The current international standard for requirements engineering processes and documentation. Replaces IEEE 830-1998. Defines requirements for system, software, and stakeholder needs specification.  
**Key Sections**: Requirements characteristics, SyRS (System Requirements Specification) and SRS (Software Requirements Specification) structure.

### IEEE 830-1998 — Recommended Practice for Software Requirements Specifications
**Organization**: IEEE Computer Society  
**Description**: Historical standard for SRS structure and quality attributes. Still referenced widely despite being superseded.

### BABOK Guide v3
**Organization**: IIBA  
**Description**: Global standard for business analysis practice. Defines knowledge areas, tasks, techniques, and competencies.  
**URL**: https://www.iiba.org/career-resources/a-business-analysis-body-of-knowledge/babok/

### Gherkin Language Specification
**Organization**: Cucumber Ltd.  
**Description**: The syntax standard for writing BDD scenarios (Feature, Scenario, Given, When, Then, And, But, Background, Scenario Outline, Examples).  
**URL**: https://cucumber.io/docs/gherkin/reference/
