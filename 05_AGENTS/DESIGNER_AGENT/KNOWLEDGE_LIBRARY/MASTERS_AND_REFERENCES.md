# MASTERS AND REFERENCES — DESIGNER AGENT

## Steve Krug
**Domain**: Web usability, user experience  
**Key Contributions**:
- Authored the most accessible and widely-read book on web usability.
- Established the "don't make me think" principle: a web page should be self-evident, obvious, and self-explanatory.
- Popularized low-cost usability testing ("hallway testing") as a continuous practice.
- Introduced the idea that users don't read — they scan — and design must account for scanning behavior.

**Primary Works**: *Don't Make Me Think* (2000; 3rd ed. 2014), *Rocket Surgery Made Easy* (2009)  
**Relevance to Designer**: Foundational usability principles, user testing methodology, information hierarchy.

---

## Don Norman
**Domain**: Human-centered design, cognitive psychology applied to design  
**Key Contributions**:
- Coined the term "user experience" in the context of design.
- Defined core design concepts: affordances (what actions an object enables), signifiers (signals that communicate affordances), feedback, mapping, and constraints.
- Popularized the concept of "Norman doors" as a metaphor for designs that confuse their users.
- Argued that design failures are almost always designer failures, not user failures.

**Primary Work**: *The Design of Everyday Things* (1988; revised ed. 2013)  
**Relevance to Designer**: Foundational design theory, affordance/signifier model, feedback loops, error design.

---

## Nielsen Norman Group (Jakob Nielsen & Don Norman)
**Domain**: UX research, usability heuristics  
**Key Contributions**:
- Jakob Nielsen defined the 10 Usability Heuristics (1994), the most widely applied usability evaluation framework.
- Produced extensive quantitative research on usability (5-user testing rule, F-pattern reading, eye tracking studies).
- Developed the heuristic evaluation method for expert review of interfaces.

**Primary Works**: Nielsen Norman Group research reports, nngroup.com articles  
**Heuristics Reference**: https://www.nngroup.com/articles/ten-usability-heuristics/  
**Relevance to Designer**: Heuristic evaluations, usability testing protocols, research-based design guidance.

---

## Luke Wroblewski
**Domain**: Mobile-first design, web forms  
**Key Contributions**:
- Coined and popularized the "Mobile First" design strategy.
- Argued that designing for the most constrained environment (mobile) produces better design decisions than retrofitting desktop designs to smaller screens.
- Published extensive research on web form design (label placement, input types, validation patterns).

**Primary Work**: *Mobile First* (2011, A Book Apart)  
**Relevance to Designer**: Mobile-first design approach, responsive design rationale, form design patterns.

---

## Brad Frost
**Domain**: Atomic Design, design systems, pattern libraries  
**Key Contributions**:
- Created Atomic Design: a systematic methodology for building design systems from fundamental building blocks.
- Developed Pattern Lab, an open-source tool for building atomic design systems.
- Advocated for a shared design language between designers and developers.

**Primary Work**: *Atomic Design* (2016, free online at atomicdesign.bradfrost.com)  
**Relevance to Designer**: Design system architecture, component hierarchy, design token strategy.

---

## Nir Eyal
**Domain**: Behavioral design, habit-forming products  
**Key Contributions**:
- Developed the Hook Model: Trigger → Action → Variable Reward → Investment.
- Analyzed the psychological mechanisms that make products habit-forming (Instagram, Twitter, gaming).
- Provided ethical frameworks for applying behavioral design responsibly.

**Primary Work**: *Hooked: How to Build Habit-Forming Products* (2014)  
**Relevance to Designer**: Engagement patterns, onboarding design, notification design, retention mechanics.

---

## Adam Wathan & Steve Schoger
**Domain**: Practical UI design, visual design for developers  
**Key Contributions**:
- Published *Refactoring UI* as a practical guide to improving UI visual quality without a design background.
- Demonstrated how small changes in spacing, typography, color, and hierarchy produce dramatically better interfaces.
- Created Tailwind CSS, a utility-first CSS framework aligned with their design principles.

**Primary Work**: *Refactoring UI* (2018, self-published at refactoringui.com)  
**Relevance to Designer**: Visual design decisions, spacing systems, color palettes, typography hierarchy.

---

## W3C Web Accessibility Initiative
**Domain**: Web accessibility standards  
**Key Contributions**:
- Published WCAG (Web Content Accessibility Guidelines), the global standard for web accessibility.
- Defined the POUR principles: Perceivable, Operable, Understandable, Robust.
- Provides conformance levels A, AA, and AAA with specific technical criteria.

**Primary Work**: WCAG 2.1 (2018), WCAG 2.2 (2023)  
**URL**: https://www.w3.org/WAI/standards-guidelines/wcag/  
**Relevance to Designer**: Accessibility compliance, inclusive design, legal requirement baseline (most jurisdictions require WCAG 2.1 AA).
