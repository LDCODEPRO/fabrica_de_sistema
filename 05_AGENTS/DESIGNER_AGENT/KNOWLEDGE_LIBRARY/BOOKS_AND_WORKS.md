# BOOKS AND WORKS — DESIGNER AGENT

## Core Books

### Don't Make Me Think (3rd Edition)
**Author**: Steve Krug  
**Year**: 2014 (3rd ed.; 1st ed. 2000)  
**Publisher**: New Riders  
**Summary**: The most readable and influential book on web usability. Krug argues that good design is self-evident — users should never have to think about how to use a page. Covers navigation, search, homepage design, user testing, and accessibility.  
**Key Concepts**: Scanning over reading, satisficing behavior, navigation conventions, usability testing with 5 users, "trunk test" for navigation clarity.

---

### The Design of Everyday Things (Revised and Expanded Edition)
**Author**: Don Norman  
**Year**: 2013 (revised; 1st ed. 1988 as *The Psychology of Everyday Things*)  
**Publisher**: Basic Books  
**Summary**: A landmark work in human-centered design. Norman uses everyday objects (doors, faucets, stoves) to explain why design failures are design problems, not user problems. Introduces the foundational vocabulary of UX: affordances, signifiers, conceptual models, and feedback.  
**Key Concepts**: Affordances, signifiers, mappings, feedback, constraints, forcing functions, human error categories.

---

### Mobile First
**Author**: Luke Wroblewski  
**Year**: 2011  
**Publisher**: A Book Apart  
**Summary**: A short, influential manifesto arguing that designers should start with mobile constraints and expand to larger screens, rather than the reverse. Mobile constraints force prioritization and clarity.  
**Key Concepts**: Progressive enhancement, content-first design, touch targets (minimum 44px), mobile constraints as a design filter.

---

### Atomic Design
**Author**: Brad Frost  
**Year**: 2016  
**Publisher**: Available free at atomicdesign.bradfrost.com  
**Summary**: Defines a systematic methodology for building design systems using five levels of hierarchy: atoms, molecules, organisms, templates, and pages. Bridges design and development with shared component vocabulary.  
**Key Concepts**: Atoms (buttons, inputs, labels), molecules (form fields = label + input + error), organisms (header = logo + nav + search), templates, pages.

---

### Hooked: How to Build Habit-Forming Products
**Author**: Nir Eyal  
**Year**: 2014  
**Publisher**: Portfolio/Penguin  
**Summary**: Describes the Hook Model — a four-phase loop (Trigger, Action, Variable Reward, Investment) that explains why some products create habits. Draws on behavioral economics and psychology. Includes an ethics framework ("The Manipulation Matrix").  
**Key Concepts**: Hook Model, internal vs. external triggers, variable rewards (tribe, hunt, self), investment and commitment, ethical design.

---

### Refactoring UI
**Authors**: Adam Wathan & Steve Schoger  
**Year**: 2018  
**Publisher**: Self-published (refactoringui.com)  
**Summary**: A practical, visual guide to improving UI design quality. Covers spacing, color, typography, shadows, borders, and component design through before/after examples. Aimed at developers who need to make design decisions without a dedicated designer.  
**Key Concepts**: 8-point spacing system, layered shadows, HSL color model, type hierarchy, visual weight, designing in grayscale first.

---

### Rocket Surgery Made Easy
**Author**: Steve Krug  
**Year**: 2009  
**Publisher**: New Riders  
**Summary**: A practical guide to running usability tests cheaply and frequently. Argues that imperfect testing done often beats perfect testing done rarely. Provides scripts, checklists, and facilitation guidance.  
**Key Concepts**: Debrief sessions, think-aloud protocol, remote testing, recruiting participants, prioritizing fixes.

---

### About Face: The Essentials of Interaction Design (4th Edition)
**Authors**: Alan Cooper, Robert Reimann, David Cronin & Christopher Noessel  
**Year**: 2014  
**Publisher**: Wiley  
**Summary**: A comprehensive reference on interaction design, covering design research, persona creation, scenario-based design, behavioral patterns, and design patterns for desktop and mobile applications.  
**Key Concepts**: Goal-directed design, personas (Cooper's original method), design scenarios, platform idioms, posture and flow.
